# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Package manager and task runner

Use **pnpm** to install dependencies (`pnpm install`). Root-level scripts (defined in the root `package.json`) are run via **npm run** and are proxied through **Turbo**. Do not use `npm install` — it will create a conflicting `package-lock.json`.

## Common commands

```bash
# Install
pnpm install

# Local development (run in separate terminals, in order)
npm run sandbox:platform       # Terminal 1 — wait until ready
npm run sandbox:todo           # Terminal 2 — waits on platform SSM param
npm run dev                    # Terminal 3 — Vite dev server at http://localhost:5173

# Build frontend
npm run build

# Amplify outputs (needed before `npm run dev` if sandbox isn't running)
npm run generate-outputs:platform
npm run generate-outputs:todo

# Bootstrap (first-time AWS setup only)
npm run bootstrap:deploy
npm run bootstrap:diff
```

## Architecture

This is a split-backend Amplify Gen 2 monorepo. The key design decision is that **auth never deploys on feature branches** — it lives in a stable platform stack, and feature backends reference it via an SSM parameter.

```
packages/platform/      ← Cognito auth. Deploy from main/release only.
packages/todo-backend/  ← AppSync data API. Deploy from any branch.
apps/todo-app/          ← React + Vite frontend.
packages/bootstrap/     ← CDK stack. Run once to create the 3 Amplify apps in AWS.
```

**How the stacks connect:**
- `platform/amplify/backend.ts` deploys auth and writes a JSON blob to SSM at `/derrops/amplify-todo/platform`.
- `todo-backend/amplify/backend.ts` reads that SSM param at synth time and calls `referenceAuth(...)` to wire its AppSync API to the existing Cognito resources — it never owns or re-deploys auth.
- The frontend imports `amplify_outputs.json` from `packages/todo-backend/` (gitignored, generated by the sandbox or CI).

**Frontend wiring (`apps/todo-app/src/`):**
- `main.tsx` — calls `Amplify.configure(outputs)` and wraps the app in `<Authenticator>`.
- `client.ts` — exports `generateClient<Schema>()` typed against the todo-backend schema. Import `client` and `TodoItem` from here, never instantiate `generateClient` elsewhere.
- TypeScript path aliases: `@todo-backend/*` → `packages/todo-backend/*`, `@platform/*` → `packages/platform/*`.

**Adding data models:** edit `packages/todo-backend/amplify/data/resource.ts`. The `Schema` type is automatically picked up by `client.ts`.

**CI/CD (amplify.yml):** Three Amplify apps correspond to the three roots. The frontend build pulls `amplify_outputs.json` from the todo-backend app at build time using `ampx generate outputs --branch $AWS_BRANCH --app-id $TODO_BACKEND_APP_ID`.
